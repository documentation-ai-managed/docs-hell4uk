## Setting Up Webhooks

Webhooks notify external services on events like doc publishes or edits. Configure them in your hell4uk dashboard under Settings > Webhooks.

<Steps>
  <Step title="Generate Webhook URL" icon="link">
    Copy the unique URL provided, e.g., `https://api.example.com/v1/webhooks/your-webhook-id`.
  </Step>
  <Step title="Select Events" icon="zap">
    Choose from `doc.updated`, `doc.published`, or `comment.added`.
  </Step>
  <Step title="Configure Secret" icon="lock">
    Add a shared secret for signature verification.
  </Step>
  <Step title="Test Endpoint" icon="play">
    Trigger a test event and verify receipt.
  </Step>
</Steps>

<CodeGroup tabs="Node.js,Python">
  ```javascript
  const crypto = require('crypto');
  function verifySignature(payload, signature, secret) {
    const hmac = crypto.createHmac('sha256', secret);
    hmac.update(payload);
    return hmac.digest('hex') === signature.replace('sha256=', '');
  }
  ```
  ```python
  import hmac
  import hashlib

  def verify_signature(payload, signature, secret):
      computed = hmac.new(
          secret.encode(), payload.encode(), hashlib.sha256
      ).hexdigest()
      return hmac.compare_digest(computed, signature.replace('sha256=', ''))
  ```
</CodeGroup>

## API Access for Custom Extensions

Access hell4uk data programmatically via REST API at `https://api.example.com/v1`.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`.
</ParamField>

<ParamField query="spaceId" param-type="string" required="true">
  Your documentation space identifier.
</ParamField>

```javascript
// Fetch docs list
const response = await fetch('https://api.example.com/v1/docs?spaceId=YOUR_SPACE_ID', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});
const docs = await response.json();
```

<Callout kind="alert">
  Store `YOUR_API_KEY` securely. Never expose it in client-side code.
</Callout>

## Exporting Documentation

Export your hell4uk docs to other formats for sharing or archiving.

<Tabs>
  <Tab title="PDF Export" icon="file-text">
    Generate print-ready PDFs from the dashboard.

    ```
    POST https://api.example.com/v1/export/pdf?spaceId=YOUR_SPACE_ID
    ```
  </Tab>
  <Tab title="Markdown" icon="code">
    Download full site as Markdown files.

    ```
    GET https://api.example.com/v1/export/md?spaceId=YOUR_SPACE_ID
    ```
  </Tab>
</Tabs>

<Expandable title="Advanced Export Options" default-open="false">
  Use query params like `?includeAttachments=true` for media files.
</Expandable>
